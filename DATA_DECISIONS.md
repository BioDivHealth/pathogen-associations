# Data Decisions

This document records current operating decisions for pathogen-host-vector
association data. It is project guidance, not a changelog. Use it before making
biological role claims, changing row-inclusion logic, or merging evidence layers
into modelling-readiness outputs.

The central rule is conservative interpretation: keep evidence layers separate
until a source-backed review justifies a role or modelling decision.

## Current Anchors

The main disease-level anchor is:

`pathogen_association_data/evidence/who_diseases/master_expansion/master_plus_who_analysis_units.csv`

It is generated by:

`scripts/associations/network_building/stages/master_plus_registry/1_2d_Master_WHO_Analysis_Unit_Bridge.R`

This table combines the disease master list, WHO analysis units, manual
name-resolution and transmission rules, and VIRION/CLOVER pathogen matching. It
is the default disease or pathogen analysis-unit registry for downstream
readiness work.

The main host-evidence backbone is:

`pathogen_association_data/evidence/who_networks/host_pathogen/master_plus_who_host_network.csv`

It is generated by:

`scripts/associations/network_building/stages/master_plus_host_network/4_2_Combine_WHO_Master_Host_Network.R`

This is a host association evidence table. It is not a host-role table.

The current WHO-only host-vector integration outputs live under:

`pathogen_association_data/evidence/who_networks/host_vector/who_only/`

These outputs are scoped to the current WHO vector-screened disease set. They
are useful for candidate generation and WHO-focused summaries, but they are not
full master-plus/readiness vector-role assignments.

Current modelling-readiness handoffs live under:

`pathogen_association_data/readiness/`

These are planning and collaborator handoff surfaces. They combine many evidence
layers and retain uncertainty; they are not final biological evidence claims.

## Key Interpretation Boundaries

Host association evidence answers:

- has this host been associated with this pathogen or disease in a source?
- what method supports the association?
- is the row high enough quality for first-pass downstream use?

It does not answer, by itself:

- is this host a reservoir, maintenance host, amplifier, or dead-end host?
- should this host define the ecological range?

Vector evidence answers:

- has this vector been implicated, observed, or tested in relation to a disease
  or pathogen?
- what evidence basis supports the row?
- is there competence evidence such as infection, transmission, mixed results,
  non-competence, or uncertainty?

It does not answer, by itself:

- is this vector the main enzootic, bridge, epidemic, or mechanical vector?
- should this vector define the ecological range?

Do not infer reservoir status from PCR/sequencing host association alone. Do not
infer vector role from host-vector observation alone.

## Source-To-Role Matrix

| Role or field | Host network | Vector screening | Host-vector sources | Competence layer | Extra evidence still needed |
| --- | --- | --- | --- | --- | --- |
| Host present in disease system | yes | no | no | no | no for association, yes for role |
| Vector implicated in transmission | no | yes | no | partial | often no for candidate use |
| Vector competence evidence | no | partial | no | yes | no for candidate use, yes for final role review |
| Vector uses host | no | no | yes | no | no for host-use evidence |
| Bridge vector candidate | partial | partial | partial | partial | yes |
| Enzootic vector candidate | partial | partial | partial | partial | yes |
| Reservoir host | partial | no | no | no | yes |
| Amplifier host | partial | no | no | no | yes |
| Incidental host | partial | no | no | no | yes |
| Dead-end host | partial | no | no | no | yes |

`yes` means the source can directly support that evidence type. `partial` means
the source can help generate candidates or provide weak support, but it is not
enough for a final role label.

## Default Filtering Rules

For disease-level modelling scope:

```r
modelling_scope_status == "include"
```

For host evidence:

```r
downstream_default_include == TRUE
```

For default disease plus host evidence:

```r
modelling_scope_status == "include" &
  downstream_default_include == TRUE
```

For role assignment:

- never use default host/vector evidence alone as final role labels
- create source-backed evidence rows first
- assign reviewed roles only in the assignment tables

For broad or aggregate rows:

- keep them for provenance and review where useful
- mark them with explicit scope fields
- do not delete them solely because they are broad
- do not use them in default modelling unless a narrower analysis unit has been
  selected or the aggregate unit is deliberately approved

## Guardrail

Do not collapse:

- host association
- host use by vectors
- vector implication
- vector competence
- ecological transmission role
- country occurrence evidence
- SDM availability

into a single binary "connected therefore important" interpretation.

The repository can generate candidate disease-host and disease-vector systems.
The main limitation is biological role assignment and disease-level modelling
readiness: deciding which evidence layers should actually constrain a disease's
range.
